\begin{DoxyItemize}
\item no\+Dict \+: There is no preceding content.
\item with\+Prefix64k \+: Table entries up to ctx-\/\texorpdfstring{$>$}{>}dict\+Size before the current blob blob being compressed are valid and refer to the preceding content (of length ctx-\/\texorpdfstring{$>$}{>}dict\+Size), which is available contiguously preceding in memory the content currently being compressed.

If dst\+Capacity \texorpdfstring{$>$}{>}= LZ4\+\_\+compress\+Bound(src\+Size), compression is guaranteed to succeed, and runs faster.

It\textquotesingle{}s not possible to append blocks together and expect a single invocation of LZ4\+\_\+decompress\+\_\+$\ast$() to decompress them together.

LZ4\+\_\+decompress\+\_\+$\ast$\+\_\+continue() \+: These decoding functions allow decompression of consecutive blocks in \char`\"{}streaming\char`\"{} mode. A block is an unsplittable entity, it must be presented entirely to a decompression function. Decompression functions only accepts one block at a time. The last 64KB of previously decoded data {\itshape must} remain available and unmodified at the memory position where they were decoded. If less than 64KB of data has been decoded, all the data must be present.

Special \+: if decompression side sets a ring buffer, it must respect one of the following conditions \+:
\begin{DoxyItemize}
\item Decompression buffer size is {\itshape at least} LZ4\+\_\+decoder\+Ring\+Buffer\+Size(max\+Block\+Size). max\+Block\+Size is the maximum size of any single block. It can have any value \texorpdfstring{$>$}{>} 16 bytes. In which case, encoding and decoding buffers do not need to be synchronized. Actually, data can be produced by any source compliant with LZ4 format specification, and respecting max\+Block\+Size.
\item Synchronized mode \+: Decompression buffer size is {\itshape exactly} the same as compression buffer size, and follows exactly same update rule (block boundaries at same positions), and decoding function is provided with exact decompressed size of each block (exception for last block of the stream), {\itshape then} decoding \& encoding ring buffer can have any size, including small ones ( \texorpdfstring{$<$}{<} 64 KB).
\item Decompression buffer is larger than encoding buffer, by a minimum of max\+Block\+Size more bytes. In which case, encoding and decoding buffers do not need to be synchronized, and encoding ring buffer can have any size, including small ones ( \texorpdfstring{$<$}{<} 64 KB).
\end{DoxyItemize}

Whenever these conditions are not possible, save the last 64KB of decoded data into a safe buffer where it can\textquotesingle{}t be modified during decompression, then indicate where this data is saved using \mbox{\hyperlink{lz4_8h_a81965fd474a946db05698f9aff385603}{LZ4\+\_\+set\+Stream\+Decode()}}, before decompressing next block. \mbox{\Hypertarget{lz4_8c_a8dbbbc837a88da1386c24d27d5cffe81}\label{lz4_8c_a8dbbbc837a88da1386c24d27d5cffe81}}

\mbox{\hyperlink{lz4_8h_aaba8ca0bd1f23633bdd834b36bb38989}{LZ4\+\_\+decompress\+\_\+safe\+\_\+partial()}} \+: Decompress an LZ4 compressed block, of size \textquotesingle{}src\+Size\textquotesingle{} at position \textquotesingle{}src\textquotesingle{}, into destination buffer \textquotesingle{}dst\textquotesingle{} of size \textquotesingle{}dst\+Capacity\textquotesingle{}. Up to \textquotesingle{}target\+Output\+Size\textquotesingle{} bytes will be decoded. The function stops decoding on reaching this objective. This can be useful to boost performance whenever only the beginning of a block is required.

\begin{DoxyReturn}{Returns}

\end{DoxyReturn}

can be \texorpdfstring{$<$}{<} target\+Output\+Size, if compressed block contains less data.
\end{DoxyReturn}
Note 2 \+: target\+Output\+Size must be \texorpdfstring{$<$}{<}= dst\+Capacity

Note 3 \+: this function effectively stops decoding on reaching target\+Output\+Size, so dst\+Capacity is kind of redundant. This is because in older versions of this function, decoding operation would still write complete sequences. Therefore, there was no guarantee that it would stop writing at exactly target\+Output\+Size, it could write more bytes, though only up to dst\+Capacity. Some \char`\"{}margin\char`\"{} used to be required for this operation to work properly. Thankfully, this is no longer necessary. The function nonetheless keeps the same signature, in an effort to preserve API compatibility.

Note 4 \+: If src\+Size is the exact size of the block, then target\+Output\+Size can be any value, including larger than the block\textquotesingle{}s decompressed size. The function will, at most, generate block\textquotesingle{}s decompressed size.

An LZ4\+\_\+stream\+\_\+t must be initialized once before usage. This is automatically done when created by \mbox{\hyperlink{lz4_8h_a62561f6ed380269f1f2d20e659cd5865}{LZ4\+\_\+create\+Stream()}}. \mbox{\hyperlink{lz4_8h_a2298f038b5c1c52e77d9222add683a0a}{LZ4\+\_\+reset\+Stream\+\_\+fast()}} is much faster than \mbox{\hyperlink{lz4_8h_a0da60953abe0734aee6a462864fd9aa3}{LZ4\+\_\+init\+Stream()}}, but is not compatible with memory regions containing garbage data.
